### 2026-03-22: CDDG 模型复现工作记录
**1. 思考与分析过程：**
- **数据集与数据加载**：仔细阅读了 `MEDG.py` 和 `MyNewDataset.py`。确认输入数据格式为 `[B, 8, 2048]`，使用 `NormalDataset` 根据 `config.py` 中配置的任务 7（TASK=7）进行源域（带标签）和目标域（无标签验证与测试）的数据装载。
- **模型结构设计 (`CDDGNet.py`)**：
  - 根据论文，模型包含四个部分：因果编码器 $E_c$、域编码器 $E_d$、解码器 $D$、分类器 $F_c$。
  - **特征提取器**：遵循要求，直接引入 `MEDGNet.py` 中现有的高复杂度纯 CNN `FeatureEncoder`（WDCNN + 多尺度残差结构），并将 $E_c$ 和 $E_d$ 实例化为两个参数独立的编码器，均输出128维特征。
  - **解码器**：设计了一个1D转置卷积（ConvTranspose1d）解码器，将合并后的 $[Z_c, Z_d]$ (256维) 逐步上采样恢复为原始的 `[B, 8, 2048]` 信号，以计算重建损失。
  - **分类器**：采用全连接层（MLP），输入因果特征 $Z_c$，输出7类故障概率。
- **损失函数实现 (`CDDG.py`)**：
  - $L_{cl}$（分类损失）：使用标准交叉熵（Cross-Entropy），仅针对源域样本的 $Z_c$ 特征计算。
  - $L_{ca}$（因果聚合损失）：实现了一个基于距离度量的损失函数。对 $Z_c$ 依据同类别聚类/异类别排斥；对 $Z_d$ 依据同域聚类/异域排斥。
  - $L_{rc}$（重建损失）：使用 MSE 比较原信号 $X$ 和解码器输出的 $\hat{X}$。
  - $L_{rr}$（冗余减少损失）：实现了 Barlow Twins 风格的去相关矩阵损失，降低 $Z_c$ 和 $Z_d$ 内部维度的冗余度，以及 $Z_c$ 与 $Z_d$ 之间的相关性。

**4. 工程实现细节的二次修复 (2026-03-22)：**
- **学习率调度器修复**：发现 `StepLR` 原本基于 Epoch 进行 step（这在长数据集中会导致很久不衰减）。为了与作者原论文“每50次迭代（iteration）衰减”的意图对齐，将 `scheduler.step()` 移入了 `for step, (...) in enumerate(source_loader):` 批次循环的内部。
- **分类损失动态域权重 (Dynamic Domain Weighting)**：补充了官方源码中非常重要的工程技巧。通过 `reduction='none'` 拿到未 reduce 的交叉熵，在当前的混合 batch 中，动态计算每个 domain 的 mean loss。然后应用公式 `1 + (domain_mean / sum_domain_mean)` 对难以分类的领域进行权重倾斜，再求平均。
- **架构确认**：仔细核对了 `CDDGNet.py` 中分类器的输入。确认 `y_pred = self.Classifier(z_c)` 严格仅使用了因果因子 `$Z_c$`，没有错误拼接 `$Z_d$`，保证了梯度的正确路由和解耦的纯粹性。
- **DataLoader多进程打印问题**：修复了在 Windows 平台下因 DataLoader `num_workers=4` 导致的多进程 `spawn` 不断重复打印 `Random seed set to: 42` 的问题，将其改为 `num_workers=0` 以保证单进程稳定输出。
- **全局迭代步数(Global Iterations)与Epoch的换算修复**：原代码使用了 `epochs=5` 配合长数据集，会导致一个 Epoch 内可能产生几百个 Batch，造成学习率过早衰减归零。修复为使用 `global_step` 变量追踪全局迭代，在到达 `max_iterations = 200` 时直接终止训练。确保与原论文中描述的“总共只训练 200 次迭代，每 50 次衰减”完全等价。
- **最终验证**：重新跑通代码，模型在严格的 200 Iterations 下迅速收敛，并且打印出了 `Iteration 50, 100, 150, 200` 的衰减日志，验证集 Accuracy 稳定。
